# Contributing to Neon Protocol IDE

We love your input! We want to make contributing to Neon Protocol IDE as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## Our Process

We use GitHub to host code, to track issues and feature requests, as well as accept pull requests.

## Getting Started for Contributors

1. Fork the repo and create your branch from `main`.
2. Run the one-click setup script (`start.bat` on Windows or `start.sh` on macOS) to set up your environment automatically.
3. If you prefer manual setup, follow the instructions in [README.md](README.md#development-setup-manual).
4. If you've added code that should be tested, add tests.
5. If you've changed APIs, update the documentation.
6. Ensure the test suite passes.
7. Make sure your code lints.
8. Issue that pull request!
